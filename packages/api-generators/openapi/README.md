# @web-loom/api-generator-openapi

OpenAPI 3.1 specification generator for Web Loom API Framework.

## Features

- Generate OpenAPI 3.1 specifications from model and route definitions
- Automatic schema generation from model fields with validation constraints
- Support for all HTTP methods (GET, POST, PUT, PATCH, DELETE, etc.)
- Path parameter extraction and documentation
- Request/response schema generation with proper status codes
- Validation schema integration (body, query, params, headers)
- Security scheme integration with per-route authentication
- Rate limiting documentation via OpenAPI extensions
- Caching configuration via OpenAPI extensions
- Comprehensive error response schemas (400, 401, 403, 404, 429, 500)
- Example generation for requests and responses
- Export to JSON or YAML format

## Installation

```bash
npm install @web-loom/api-generator-openapi
```

## Usage

### Basic Usage

```typescript
import { OpenAPIGenerator } from '@web-loom/api-generator-openapi';
import { ModelRegistry, RouteRegistry } from '@web-loom/api-core';

// Create generator
const generator = new OpenAPIGenerator({
  title: 'My API',
  version: '1.0.0',
  description: 'API documentation',
  servers: [
    { url: 'https://api.example.com', description: 'Production' },
    { url: 'http://localhost:3000', description: 'Development' },
  ],
});

// Register models
const models = ModelRegistry.getAll();
generator.registerModels(models);

// Register routes
const routes = RouteRegistry.getAll();
generator.registerRoutes(routes);

// Generate specification
const spec = generator.generate();

// Export as JSON
const json = generator.toJSON();
console.log(json);

// Export as YAML
const yaml = generator.toYAML();
console.log(yaml);
```

### With Security Schemes

```typescript
const generator = new OpenAPIGenerator({
  title: 'Secure API',
  version: '1.0.0',
  securitySchemes: {
    bearerAuth: {
      type: 'http',
      scheme: 'bearer',
      bearerFormat: 'JWT',
    },
    apiKey: {
      type: 'apiKey',
      in: 'header',
      name: 'X-API-Key',
    },
  },
  security: [
    { bearerAuth: [] },
  ],
});
```

### With Tags and External Documentation

```typescript
const generator = new OpenAPIGenerator({
  title: 'My API',
  version: '1.0.0',
  tags: [
    { name: 'Users', description: 'User management endpoints' },
    { name: 'Posts', description: 'Blog post endpoints' },
  ],
  externalDocs: {
    description: 'Full documentation',
    url: 'https://docs.example.com',
  },
});
```

### With Route Validation Schemas

```typescript
import type { RouteDefinition } from '@web-loom/api-core';

const routes: RouteDefinition[] = [
  {
    path: '/users/:id',
    method: 'GET',
    handler: async () => new Response(),
    validation: {
      params: {
        id: { type: 'string', format: 'uuid' },
      },
      query: {
        include: { type: 'string', enum: ['posts', 'comments'] },
      },
    },
    metadata: {
      description: 'Get user by ID with optional related data',
      tags: ['Users'],
    },
  },
];

generator.registerRoutes(routes);
```

### With Rate Limiting and Caching

```typescript
const routes: RouteDefinition[] = [
  {
    path: '/users',
    method: 'GET',
    handler: async () => new Response(),
    rateLimit: {
      limit: 100,
      window: 60000, // 1 minute in milliseconds
    },
    cache: {
      ttl: 300, // 5 minutes in seconds
      perUser: false,
    },
    metadata: {
      description: 'List all users with rate limiting and caching',
      tags: ['Users'],
    },
  },
];

generator.registerRoutes(routes);
// The generated spec will include x-rate-limit and x-cache extensions
```

## Configuration Options

### OpenAPIGeneratorOptions

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `title` | `string` | `'Web Loom API'` | API title |
| `version` | `string` | `'1.0.0'` | API version |
| `description` | `string` | `'API generated by Web Loom Framework'` | API description |
| `servers` | `OpenAPIServer[]` | `[{ url: 'http://localhost:3000' }]` | Server URLs |
| `contact` | `OpenAPIContact` | `undefined` | Contact information |
| `license` | `OpenAPILicense` | `undefined` | License information |
| `securitySchemes` | `Record<string, OpenAPISecurityScheme>` | `{}` | Security schemes |
| `security` | `OpenAPISecurityRequirement[]` | `[]` | Global security requirements |
| `tags` | `OpenAPITag[]` | `[]` | Tags for grouping operations |
| `externalDocs` | `OpenAPIExternalDocs` | `undefined` | External documentation |
| `includeExamples` | `boolean` | `true` | Include examples in schemas |
| `includeDeprecated` | `boolean` | `true` | Include deprecated routes |

## Schema Mapping

The generator automatically maps Web Loom field types to OpenAPI schema types:

| Field Type | OpenAPI Type | OpenAPI Format |
|------------|--------------|----------------|
| `string` | `string` | - |
| `number` | `number` | - |
| `boolean` | `boolean` | - |
| `date` | `string` | `date-time` |
| `uuid` | `string` | `uuid` |
| `enum` | `string` | - |
| `json` | `object` | - |
| `array` | `array` | - |
| `decimal` | `number` | `double` |

## Validation Constraints

Validation rules from model fields are automatically converted to OpenAPI constraints:

- `required` → `required` array
- `min` → `minimum` / `minLength`
- `max` → `maximum` / `maxLength`
- `pattern` → `pattern`
- `email` → `format: 'email'`
- `url` → `format: 'uri'`
- `uuid` → `format: 'uuid'`
- `enum` → `enum` array

## Response Schemas

The generator automatically creates comprehensive response schemas for each endpoint:

### Success Responses

- **GET (single)**: Returns the model schema with example
- **GET (list)**: Returns paginated response with data array and pagination object
- **POST**: Returns created resource with 201 status
- **PUT/PATCH**: Returns updated resource with 200 status
- **DELETE**: Returns success message with 200 status

### Error Responses

All endpoints include standardized error responses:

- **400 Bad Request**: Validation errors with field-level details
- **401 Unauthorized**: Authentication required (if route has auth)
- **403 Forbidden**: Insufficient permissions (if route has roles)
- **404 Not Found**: Resource not found (for single resource endpoints)
- **429 Too Many Requests**: Rate limit exceeded (if route has rate limiting)
- **500 Internal Server Error**: Unexpected server errors

Each error response includes:
- `code`: Machine-readable error code
- `message`: Human-readable error message
- `details`: Additional context (e.g., validation field errors)
- `timestamp`: ISO 8601 timestamp
- `requestId`: UUID for request tracing

## OpenAPI Extensions

The generator supports custom OpenAPI extensions for framework-specific features:

### Rate Limiting Extension

```yaml
x-rate-limit:
  limit: 100
  window: 60000
  windowUnit: milliseconds
```

### Caching Extension

```yaml
x-cache:
  ttl: 300
  ttlUnit: seconds
  perUser: false
```

## API Reference

### OpenAPIGenerator

#### Methods

##### `registerModel(model: ModelDefinition): void`

Register a single model definition.

##### `registerModels(models: ModelDefinition[]): void`

Register multiple model definitions.

##### `registerRoute(route: RouteDefinition): void`

Register a single route definition.

##### `registerRoutes(routes: RouteDefinition[]): void`

Register multiple route definitions.

##### `generate(): OpenAPISpec`

Generate the complete OpenAPI specification.

##### `toJSON(pretty?: boolean): string`

Export specification as JSON string. Set `pretty` to `false` for minified output.

##### `toYAML(): string`

Export specification as YAML string.
